# PyQt5 可视化界面使用说明

本项目提供完整的 PyQt5 可视化检测界面，入口为：

```bash
python pyqt_detect.py
```

界面名称为 **水面漂浮物智能检测系统 V1.0**，采用三分栏布局：左侧参数配置，中间实时预览，右侧统计和日志。

## 1. 支持功能

- 模型权重选择和自动扫描 `weights/`、`runs/`、当前目录下的 `.pt` 文件。
- 支持 CPU / CUDA0 / CUDA1 设备选择。
- 支持置信度、NMS IoU、输入尺寸、框线粗细、字体大小调节。
- 支持单张图片、图片文件夹、本地视频、USB 摄像头、RTSP 网络流。
- 支持 4×6 网格分块推理，25% 重叠，适合高分辨率小目标检测。
- 支持保存带框图片、目标裁剪图、JSON 元数据、CSV 统计表、标注视频。
- 支持 ByteTrack 跟踪开关，视频和流模式可显示跟踪 ID。
- 支持实时进度条、推理耗时、统计面板和运行日志。

已按项目需求剔除 SRT 和大疆专属解析内容。

## 2. 界面区域

### 顶部导航栏

- 状态灯：绿色为空闲，蓝色为推理中，红色为报错。
- 开始检测：启动当前任务。
- 暂停：暂停或继续后台任务。
- 终止：停止当前检测任务。
- 打开输出目录：打开 `output_dataset` 或你选择的输出路径。
- FPS / 耗时：显示任务过程中的速度和单帧推理耗时。

### 左侧配置面板

1. 模型加载：选择 `.pt` 权重、设备、置信度、NMS、输入尺寸，并加载模型。
2. 数据源输入：选择图片、图片文件夹、视频、摄像头或 RTSP 地址。
3. 可视化参数：调节检测框、文字、置信度、网格线和裁剪保存。
4. 批量输出：设置输出目录和导出格式。
5. 高级推理：设置线程数、FP16、跳帧、ByteTrack 跟踪。

### 中间预览区

显示实时检测画面，保持等比例缩放，不拉伸画面。底部展示当前帧号、总帧数、推理耗时和当前目标数量。

### 右侧统计和日志

- 总检测目标数量。
- 当前画面目标数量。
- 最大和最小目标面积。
- 各类别累计数量。
- 运行日志、错误提示、导出记录。

## 3. 使用流程

1. 启动界面：

```bash
python pyqt_detect.py
```

2. 在“模型加载”中选择模型权重，例如：

```text
runs/detect/train/weights/best.pt
```

3. 点击“加载模型”。
4. 在“数据源输入”中选择输入模式和路径。
5. 根据需要调整置信度、NMS、输入尺寸等参数。
6. 选择导出格式和输出目录。
7. 点击“开始检测”。
8. 在中间预览区查看检测画面，在右侧查看统计和日志。

## 4. 输出目录结构

默认输出目录为：

```text
output_dataset/
  images/   # 带框图片
  crops/    # 目标裁剪图
  json/     # JSON元数据
  videos/   # 标注视频
  detections.csv
```

## 5. 参数建议

水面漂浮物通常较小，建议：

```text
conf: 0.15~0.30
imgsz: 800 或 1280
NMS IoU: 0.45
启用分块推理: 高分辨率图片/视频建议开启
跳帧: 长视频可设置为 2~5
```

如果误检多，提高 conf；如果漏检多，降低 conf 或启用 P2 模型和分块推理。

## 6. 常见问题

### 模型加载失败

检查 `.pt` 文件是否存在，或先使用命令行训练得到 `best.pt`。

### 摄像头打不开

本地摄像头通常填写 `0` 或 `1`。RTSP 流需要填写完整地址。

### 显存不足

降低输入尺寸，关闭分块推理，关闭 FP16 或切换到 CPU。

### 导出视频失败

确认输出目录有写入权限，并使用常见视频格式如 mp4。
